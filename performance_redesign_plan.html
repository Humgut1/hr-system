성과관리 체계 개편

Workday · Lattice · 레몬베이스 리서치 + 현행 화면 실측 감사 기반 — Core~Growth(30~200인 실무) 기준 재설계.
9박스·Potential·Talent Card는 Enterprise 격리(v1.2.13) 유지, 이 기획에서 다루지 않음.

1문제의식 — 왜 다시 개편하나

v1.1.0에서 뼈대(상태머신·목표 승인·이의제기·단계 게이팅)는 완성. 그러나 화면이 실무 사용을 버티지 못함 (2026-07-16 데모 실측)

화면실측 문제
관리자/매니저 성과 홈50명 × 4목표 = 200개 목표 카드가 한 페이지에 평면 나열. 사람 단위 요약이 없어 "누구 평가가 밀렸는지" 파악 불가. 한 사람을 처리하려면 스크롤로 4곳을 찾아다녀야 함
캘리브레이션153행 플랫 테이블 + 행마다 라디오 9개. 팀 단위 진행 개념이 없어 실제 캘리브레이션 회의(팀별 조정)를 지원 못 함. 등급 분포 가이드가 확정 전에 안 보임
게이팅 누락잠재력(Potential) 배너·컬럼이 growth 요금제에 노출 — saas_plan §3 위반 (Enterprise 격리 대상)
직원 성과목표·자기평가·다면·결과가 탭으로 병렬 — "지금 단계에서 내가 할 일"이 화면 구조로 드러나지 않음
사이클 운영단계별 마감일이 없고, 미제출자 리마인드 수단이 없음 (HR이 카톡으로 독촉하는 실무 공백)
결과 공개등급이 덜렁 공개됨. 국내 실무 필수인 평가 면담 → 확인 절차 없음
한 줄 요약: 데이터 모델은 사람 중심인데 화면은 목표 중심이라 운영이 안 된다.

2벤치마크 요약

Workday

전 과정이 인박스 태스크 기반: 자기평가 제출 → 매니저에게 "Complete Manager Evaluation" 태스크 도착 → 캘리브레이션 → 면담 뒤 Acknowledgement(직원 확인) 단계로 종료. 각자 화면엔 "내 할 일"만 온다.

Lattice

① 사이클 셋업 마법사 — 대상·평가유형·질문·기간·결과 공개 범위를 한 흐름에 설정 ② 매니저 대시보드는 팀 진행률·미완료 현황 중심 ③ 결과는 "패킷"으로 묶어 공개 항목을 어드민이 선택 ④ 캘리브레이션 워크플로우 내장.

레몬베이스 국내 30~200인 표준

① 리뷰 생성 시 대상자/평가유형(셀프·동료·상향·하향)/템플릿/점수 환산/참고정보 설정, 프리셋 4종(신규입사자·360·업적·종합) ② 평가 작성 화면 우측에 피평가자의 셀프 평가·목표가 참고 패널로 표시 ③ 결과 공개 범위를 평가 유형별로 설정 ④ 평가 패턴 분석으로 불성실 평가 감지.

공통 패턴 3가지 — (a) 사람 단위 뷰 + 할 일 중심  (b) 평가 작성 시 컨텍스트(목표·자기평가) 동시 표시  (c) 공개 범위·마감·리마인드가 운영 기능으로 내장

3설계 원칙

  • 사람 단위 > 목표 단위 — 매니저·HR 화면의 기본 행은 '직원', 목표는 그 안의 상세.
  • 단계가 화면을 결정 — 사이클 단계에 따라 각 역할 화면이 "지금 할 일"만 전면 노출.
  • 기존 뼈대 재사용 — 상태머신·목표 승인·이의제기·다면 토글·하향 제한 로직 그대로. DB 변경 최소화.
  • Enterprise 배제 — 9박스·Potential·Talent Card는 이 플로우에 등장하지 않음.

4-D. 사이클 운영 (HR)

  • 단계별 마감일(선택 입력) — 스텝퍼에 D-day 표시, 경과 시 경고 배지
  • 미제출자 리마인드 — 단계별 미완료자 목록 + [리마인드 발송] (인앱 알림 일괄, 기존 add_notification 재사용)
  • 단계 전환 체크리스트 — "매니저 평가 미완료 12명" 경고 후 진행 (이의 미처리 시 종료 차단은 기존 유지)
  • 결과 확인(acknowledge) — 면담 후 직원이 [결과 확인] 클릭, HR은 확인률 추적 (Workday Acknowledgement 패턴)
