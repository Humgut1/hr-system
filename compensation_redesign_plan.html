1문제의식 — 현행 실측 (2026-07-16 데모 감사)

화면실측 문제
급여 운영 탭153명 플랫 테이블(부서순)에 검색·필터 없음. 기본급 0원(급여 미설정) 직원이 그대로 섞여 노출. 연/월 선택 + [급여 생성] 버튼 + 명단이 전부 — "이번 달 정산이 어디까지 진행됐나"가 화면 구조로 안 보임
검토 단계 부재초안 생성 → 바로 확정·발송. 전월 대비 얼마나 달라졌는지, 이상값은 없는지 검토할 화면이 없음 — 실무에서 가장 사고가 잦은 지점(중도입사 일할, 수당 급증, 누락)
연봉 조정(인상)일괄 인상 = % 입력 → 즉시 적용, 되돌리기 불가. 실무의 "조정안 작성 → 시뮬레이션 → 검토 → 적용일에 반영 → 통보" 흐름이 없음. 적용일 지정 불가(항상 즉시)
게이팅 잔여 누락Compa-Ratio가 직원 테이블 컬럼 + 분석 탭 분포 카드에 growth 요금제로 노출 — saas_plan §3 위반 (R2에서 KPI 카드만 대체하고 이 2곳을 놓침)
보상 분석 탭Compa 분포(Enterprise 대상)뿐. Core 고객이 실제로 궁금한 월 인건비 추이·부서별 인건비가 없음. 일괄 인상 폼의 부서 드롭다운은 부문~팀 49개 평면 나열
한 줄 요약: 급여는 매월 도는 사이클 업무인데 화면은 명단+버튼이고, 연봉 조정은 절차 없이 즉시 반영된다.

2벤치마크 요약

flex 급여정산 국내 표준

월 정산을 단계형 위저드로 진행: 대상자 확인 → 근태 연동 → 지급/공제 확인 → 단계별 확정·잠금(자물쇠) → 급여대장 다운로드 → 정산 완료 후 명세서 공개 설정(공개일·시각·알림). 확정한 단계는 해제 전까지 수정 불가. 연봉테이블→정산→명세 발급을 하나의 급여 사이클로 묶음.

Workday Compensation Review

연 1회 조정은 워크시트(제안 그리드)에서: 컨텍스트(현재 연봉·등급·가이드라인)와 입력 필드(제안 인상률)를 한 그리드에. 제안 → 승인 → effective date에 base pay 자동 반영. 제안 단계에서는 실제 급여가 바뀌지 않음.

TalentCore 기존 자산 (재사용)

급여 2단계(초안→확정·발송+이메일, v1.2.5) · 등급별 인상률 테이블(성과 R1 연동) · salary_history · 미래 발령 예약 반영 프레임(personnel_actions) · 감사 로그.

공통 패턴 — (a) 월 급여는 단계형 사이클(생성→검토→확정)로  (b) 연봉 조정은 "제안↔반영 분리 + 적용일 지정"  (c) 검토 화면엔 비교 컨텍스트(전월/현재 연봉)를 함께

3설계 원칙

  • 월 급여 = 사이클 — 급여 운영 탭을 "이번 달 정산 보드"로: 단계 스텝퍼가 화면을 이끈다.
  • 조정은 제안↔반영 분리 — 연봉 조정안(draft)은 몇 번이고 고치고, 반영은 적용일에 한 번.
  • 기존 뼈대 재사용 — payslips.status(초안/확정), salary_history, 등급 연동 인상률, 이메일 발송 그대로.
  • Enterprise 배제 — Salary Band·Compa·Merit Matrix·ACR은 등장하지 않음 (게이팅 누락 2곳 수리 포함).
