How It Works

이벤트 하나로 3개 서비스가 동시에 작동

TalentCore에서 직원을 등록하거나 퇴직 처리하면, 연결된 서비스들이 자동으로 반응합니다.

💬

Slack 워크스페이스 자동화

입퇴사 알림 · 채널 배정 · 환영 메시지 · 계정 비활성화

🟠 트리거 1

신규 직원 등록

TalentCore에서 직원 정보 저장 완료 시

⟶
자동 실행되는 액션
✉️
Slack 워크스페이스 초대 이메일 발송
#️⃣
소속 부서 채널에 자동 추가 (예: #개발팀, #마케팅)
👋
#general 채널에 입사 소개 메시지 자동 게시
🔴 트리거 2

퇴직 처리 확정

HR 최종 승인으로 퇴직 처리 완료 시

⟶
자동 실행되는 액션
🔒
Slack 계정 즉시 비활성화 (채널 접근 차단)
📢
퇴직 알림 (내부 HR 채널 고지)
🔵 트리거 3

부서 이동 발령

인사발령으로 부서 변경 처리 시

⟶
자동 실행되는 액션
#️⃣
새 부서 채널에 자동 추가
🔔
이동 알림 DM 발송

📝
Task: 보안 서약서 서명, 교육, 목표 설정 등 +5개
🔴 트리거 2

퇴직 처리 확정

Integration Roadmap

지금 되는 것 vs 앞으로 추가할 것

Phase 1 — 완료 ✓

기본 이벤트 연동

  • 입사 시 Slack 초대 + 채널 배정
  • 입사 시 환영 메시지 자동 게시
  • 입사 시 Jira 온보딩 Epic 생성
  • 퇴직 시 Slack 계정 비활성화
  • 퇴직 시 Jira 오프보딩 Epic 생성
  • 입사 시 Confluence 팀/프로필 페이지 생성
Phase 2 — 예정

성과·근태 연동

  • 성과 목표 설정 시 Jira 목표 이슈 생성
  • 캘리브레이션 완료 시 Slack DM 알림
  • 연장근무 승인 시 매니저에게 Slack 알림
  • 퇴직 신청 시 오프보딩 태스크 자동 생성
  • Confluence 조직도 페이지 자동 업데이트
Phase 3 — 검토 중

양방향 동기화

  • Jira 이슈 완료 → TalentCore 온보딩 체크 자동 업데이트
  • Slack 상태 변경 → 근태 자동 반영
  • Confluence 페이지 수정 → 프로필 동기화
  • Jira 프로젝트 완료율 → 성과 대시보드 연동
