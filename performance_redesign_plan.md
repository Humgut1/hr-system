# 성과관리 체계 개편 기획서 (R1)

> 작성일: 2026-07-16 · 승헌씨 승인 대기
> 근거: Workday·Lattice·레몬베이스 리서치 + 현행 화면 UX 감사 (데모 실브라우저)
> 전제: R2(v1.2.13)에서 확정된 요금제 격리 유지 — 9박스·Potential·Talent Card는 Enterprise, 이 기획은 **Core~Growth(30~200인 실무)** 기준

---

## 1. 문제의식 — 왜 다시 개편하나

v1.1.0 재개편으로 **뼈대(상태머신·목표 승인·이의제기·단계 게이팅)는 완성**됐다. 그러나 화면이 실무 사용을 버티지 못한다:

| 화면 | 실측 문제 (2026-07-16 데모 감사) |
|---|---|
| 관리자/매니저 성과 홈 | 50명 × 4목표 = **200개 목표 카드가 한 페이지에 평면 나열**. 사람 단위 요약이 없어 "누구 평가가 밀렸는지" 파악 불가. 진행률·평가 버튼이 목표 단위로 흩어져 한 사람을 처리하려면 스크롤로 4곳을 찾아다녀야 함 |
| 캘리브레이션 | **153행 플랫 테이블 + 행마다 등급 라디오 9개**. 부서 필터·팀 단위 진행 개념이 없어 실제 캘리브레이션 회의(팀별로 모여서 조정)를 화면이 지원 못 함. 등급 분포 가이드(강제 배분 대비 현재 분포)가 확정 전에 안 보임 |
| 캘리브레이션 (게이팅 누락) | 잠재력(Potential) 배너·컬럼이 **growth 요금제에 그대로 노출** — saas_plan §3 위반 (Enterprise 격리 대상) |
| 직원 성과 | 투두 위젯은 있으나 목표·자기평가·다면·결과가 탭으로 병렬 — "지금 단계에서 내가 할 일"이 화면 구조로 드러나지 않음 |
| 사이클 운영 | 단계 전환은 되지만 **단계별 마감일이 없고**, 미제출자 리마인드 수단이 없음 (HR이 카톡으로 독촉하는 실무 공백) |
| 결과 공개 | 등급이 덜렁 공개됨. 국내 실무 필수인 **평가 면담 → 확인** 절차가 없음 |

한 줄 요약: **데이터 모델은 사람 중심인데 화면은 목표 중심**이라 운영이 안 된다.

## 2. 벤치마크 요약

**Workday** — 전 과정이 **인박스 태스크 기반**: 자기평가 제출 → 매니저에게 "Complete Manager Evaluation" 태스크 도착 → 캘리브레이션 참여 후 → 면담 뒤 **Acknowledgement(직원 확인)** 단계로 종료. 각자 화면엔 "내 할 일"만 온다.

**Lattice** — ①사이클 셋업 마법사(대상·평가유형·질문·기간·**결과 공개 범위**를 한 흐름에 설정) ②매니저 대시보드: 팀 진행률·미완료 현황·요약 중심 ③결과는 "패킷(deliverable)"으로 묶어 공개 항목을 어드민이 선택 ④캘리브레이션 워크플로우 내장.

**레몬베이스** (국내 30~200인 표준) — ①리뷰 생성 시 대상자/평가유형(셀프·동료·상향·하향)/템플릿/점수 환산/참고정보를 설정, **프리셋 4종**(신규입사자·360·업적·종합) 제공 ②**평가 작성 화면 우측에 피평가자의 셀프 평가·목표가 참고 패널로 표시** ③결과 공개 범위를 평가 유형별로 설정(예: 하향 2차는 비공개) ④평가 패턴 분석으로 불성실 평가 감지.

공통 패턴 3가지: **(a) 사람 단위 뷰 + 할 일 중심**, (b) **평가 작성 시 컨텍스트(목표·자기평가) 동시 표시**, (c) **공개 범위·마감·리마인드가 운영 기능으로 내장**.

## 3. 설계 원칙

1. **사람 단위 > 목표 단위** — 매니저·HR 화면의 기본 행은 '직원'이고, 목표는 그 안의 상세다.
2. **단계가 화면을 결정** — 사이클 단계에 따라 각 역할 화면이 "지금 할 일"만 전면에 노출.
3. **기존 뼈대 재사용** — 상태머신·목표 승인·이의제기·다면 토글·하향 제한 로직은 그대로. DB 변경 최소화.
4. **Enterprise 배제** — 9박스·Potential·Talent Card는 이 플로우에 등장하지 않음(게이팅).

## 4. 화면 재설계

### 4-A. 매니저/HR — "팀 성과" (핵심)
- 상단: 사이클 스텝퍼 + 단계 마감 D-day + **내 처리 현황 요약** ("평가 대기 12명 · 목표 승인 대기 3명")
- 본문: **사람 단위 테이블** — 이름/부서/목표 n개(가중치 합)/진행률 평균/자기평가 상태/내 평가 상태/등급(확정 후)/[처리] 버튼. 기존 검색·미완료 필터(v1.2.12) 유지.
- **[처리] → 팀원 상세 패널**(슬라이드 인 또는 전용 페이지): 그 사람의 목표 전체 + 진행률 + 자기평가 코멘트 + 내 평가 입력폼이 **한 화면**에. 목표별 점수 매기고 저장하면 목록으로 복귀, 다음 미처리자로 이동 버튼.
- 목표 승인(goal 단계)도 같은 패널에서: 제출된 목표 일괄 검토 → 확정/반려.

### 4-B. 직원 — "내 성과" 단계별 단일 화면
- 탭 병렬 대신 **단계 스위칭 레이아웃**: 상단 스텝퍼 아래 본문이 현재 단계 콘텐츠로 교체
  - 목표 수립: 목표 작성/제출 (반려 사유 표시 포함)
  - 진행 중: 진행률 업데이트 카드
  - 평가 진행: 자기평가 폼 (+ 다면 대상이면 "작성할 동료평가 n건" 리스트)
  - 결과 공개: 등급 + 매니저 총평 + (포함 시) 다면 요약 + **[결과 확인] 버튼(면담 확인, acknowledge)** + 이의신청
- 지난 사이클 결과는 하단 아코디언으로.

### 4-C. 평가 작성 화면 — 참고 패널 (레몬베이스 패턴)
- 매니저 평가·다면평가 작성 화면을 **좌: 평가 폼 / 우: 참고 패널**(피평가자 목표·진행률·자기평가 코멘트·지난 등급)로 재구성. 이동 없이 근거를 보며 작성.

### 4-D. 캘리브레이션 — 팀 단위 진행
- 좌측: **부서(팀) 목록 + 팀별 확정률** → 선택한 팀만 본문에 표시
- 본문 상단: **등급 분포 바** — 현재 확정 분포 vs 권장 분포(회사 설정의 배분 기준) 실시간 비교
- 행: 종합점수 내림차순 정렬, 자기/동료/매니저 점수 + AI 제안 등급 + 등급 선택(하향 시 사유 필수 — 기존 로직) 
- 잠재력(Potential) 배너·컬럼 **Enterprise 게이팅** (누락 수리)
- 전체 요약 뷰(전 부서 분포 히트맵)는 확정 마감 전 점검용으로 별도 탭.

### 4-E. 사이클 운영 (HR)
- 단계별 **마감일(선택 입력)** — 스텝퍼에 D-day 표시, 지나면 경고 배지
- **미제출자 리마인드**: 단계별 미완료자 목록 + [리마인드 발송] (인앱 알림 일괄, 기존 add_notification 재사용)
- 단계 전환 시 체크리스트 강화: "매니저 평가 미완료 12명" 경고 후 진행 (이의 미처리 시 종료 차단은 기존 유지)

## 5. DB 변경 (최소)

| 변경 | 내용 |
|---|---|
| `performance_cycles` | `goal_deadline`, `review_deadline` DATE 컬럼 추가 (NULL 허용 — 마감 미설정 운영 가능) |
| `calibration_results` | `acknowledged_at` TIMESTAMP 추가 (결과 확인) |
| 신규 테이블 없음 | 리마인드는 notifications 재사용, 나머지는 화면 재구성 |

## 6. 구현 순서 (한 버전 = 한 작업)

| 버전 | 범위 | 비고 |
|---|---|---|
| v1.3.0 | **R1-A 매니저 '팀 성과' 재설계** — 사람 단위 테이블 + 팀원 상세 처리 패널(목표 승인+평가 통합) | 최대 임팩트, 여기부터 |
| v1.3.1 | **R1-B 직원 '내 성과' 단계별 화면** + 평가 작성 참고 패널(4-C) | |
| v1.3.2 | **R1-C 캘리브레이션 팀 단위 재설계** + Potential 게이팅 수리 | |
| v1.3.3 | **R1-D 사이클 운영** — 마감일·리마인드·전환 체크리스트 + 결과 확인(acknowledge) | DB 마이그레이션 포함 |

각 버전마다 기존 릴리즈 워크플로우(스윕→커밋→태그→릴리즈→배포) 적용.
