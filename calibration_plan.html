📚 캘리브레이션이란?
캘리브레이션(Calibration)은 각 매니저/부서에서 올라온 개인 평가 점수를 전사 기준으로 조정하는 과정입니다.
A 팀 매니저는 80%를 "우수"로 주고, B 팀 매니저는 20%만 "우수"로 주면 — 같은 성과라도 어느 팀에 있느냐에 따라 등급이 달라지는 불공정이 생깁니다.
캘리브레이션은 이 편차를 잡아주는 공정성 조율 세션입니다.
2026년 글로벌 트렌드
트렌드내용이유
강제 분포 폐지 S 10% / A 20% 식 쿼터제 → 지양 매니저 간 경쟁 유발, 실제 성과 왜곡
데이터 기반 세션 목표 달성률 + 360도 피드백 + 재직기간 동시 표시 직관이 아닌 근거로 등급 결정
분포 이상 자동 감지 특정 부서 등급이 전사 평균과 크게 다르면 경고 편향·후광효과 사전 차단
편향 모니터링 성별·연차·부서별 등급 분포 분석 특정 그룹 차별 방지
HR 확정 워크플로우 매니저 → HR 검토 → 최종 확정 단계 분리 책임 소재 명확화

🇰🇷 한국 스타트업 실무 현황
실제로 많이 하는 방식
· 매니저가 각자 평가 → HR이 엑셀로 모아서 분포 확인
· HR이 이상 부서 매니저와 1:1 면담으로 조율
· 임원 리뷰 → 최종 확정
· 200인 스타트업은 대부분 엑셀 + 구글시트로 진행
주요 고충
· 매니저마다 기준 달라서 형평성 문제
· 전사 분포 보려면 수작업으로 집계
· 조정 이력·근거 기록 안 남음
· 직원에게 결과 어떻게 전달할지 막막
💡 TalentCore가 해결해야 할 것
엑셀로 하던 캘리브레이션을 시스템 안으로 — 점수 자동 집계, 분포 시각화, 등급 수동 조정 + 이력 기록, 직원에게 공개
🔍 현재 TalentCore 캘리브레이션 상태
❌ 현재 상태: 페이지 없음
· generate_calibration_summary() 함수는 존재 (규칙 기반 텍스트 생성)
· calibration_results 테이블 존재 (final_grade, note, decided_by)
· 하지만 캘리브레이션 실행 페이지(/performance/calibration)가 없음
· 성과 페이지의 "캘리브레이션 시작" 버튼이 peer_assignments로 잘못 연결됨
· 직원이 본인 결과를 볼 방법 없음
현재 DB 테이블 — calibration_results
calibration_results
  id, cycle_id, user_id
  final_grade TEXT -- S/A/B/C/D
  note TEXT -- HR 메모
  decided_by INTEGER -- 확정한 Admin user_id
  decided_at DATETIME

❌ 없는 컬럼: self_avg, peer_avg, mgr_avg, summary_text, is_shared
⚠️ 직원 성과 페이지의 "평가 결과" 탭이 calibration_results를 조회하는데,
self_avg·peer_avg·mgr_avg·summary_text 컬럼이 없어서 결과가 아무것도 안 나올 수 있습니다.
