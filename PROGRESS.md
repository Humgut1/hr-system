## 완료된 작업

### 1단계 — 프로젝트 세팅 + 로그인 ✅
- Flask 앱 구조, SQLite DB, 로그인/로그아웃/세션
- Workday + Greenhouse 스타일 UI
- 역할별 대시보드 4종

### 2단계 — 공통 화면 + 인력 관리 ✅
- 공지사항 목록 / 상세 / 작성 / 수정
- 조직도 (부서 계층 시각화)
- 직원 관리 (목록/등록/수정/퇴직처리)
- 부서·직급 관리
- 에러 페이지 (403, 404)

### 3단계 — 근태 관리 ✅
- 내 근태 현황 (연차 잔여/사용 현황, 신청 내역)
- 근태 신청 (연차/반차/병가/재택/외출)
- 근태 신청 취소
- 근태 승인/반려 (Manager/Admin)
- 근태 캘린더 (월별, 이전/다음 월 이동)

### 4단계 — 급여 · 증명 ✅
- 급여명세서 목록 + 상세 조회 (인쇄 최적화)
- 근로기준법 §48 준수: 모든 공제 항목 명세 표시
- 4대보험 2026년 요율 자동 계산
  - 국민연금 4.5%, 건강보험 3.545%, 장기요양(건보×12.95%), 고용보험 0.9%
- 소득세 / 지방소득세 자동 계산
- 비과세 항목 (식대 20만, 교통비 20만) 소득세법 시행령 §12 반영
- 재직증명서 발급 (인쇄/PDF, 직인 포함)
- 경력증명서 발급 (인쇄/PDF, 직인 포함)
- 급여 관리 Admin 화면 (기본급/수당 설정, 월별 명세서 생성)

### 5단계 — 성과 관리 ✅
- 평가 주기 관리 (active/closed)
- 성과 목표 등록 (KPI/OKR, 가중치, SMART 가이드)
- 5점 척도 평가 입력/수정
- Admin/Manager: 팀원별 그룹 뷰, 평균 점수 표시
- Employee: 본인 목표 + 평가 결과 조회

---

### 6단계 — 채용 관리 ✅
- 채용 공고 목록 / 등록 / 수정 (draft·open·closed 상태)
- 공고 상세 + 지원자 직접 등록 (이름/이메일/경로/메모)
- 지원자 파이프라인 칸반 보드 (8단계: 지원접수→입사확정/불합격)
- 지원자 상세: 단계 변경 + 변경 이력 타임라인
- Admin/Recruiter 전용 접근 (employee·manager 403)

### 7단계 — 대시보드 개선 + 프로필 ✅
- Admin 대시보드: 실데이터 연결 (근태 대기 건수, 진행중 공고, 전체 지원자)
- Admin 대시보드: 채용 현황 위젯 + 공지 작성 버튼 추가
- Manager 대시보드: 근태 승인 대기 목록 + 인라인 승인/반려 버튼 + 팀 성과 요약
- Employee 대시보드: 실 연차 현황 (법정 계산) + 최근 신청 내역 + 빠른 메뉴
- Recruiter 대시보드: 실 채용 지표 + 최근 지원자 목록
- 알림 뱃지: 최근 3일 공지 수 동적 표시 (base.html)
- 프로필 설정 페이지 (`/profile`): 연락처 수정 + 비밀번호 변경

### 8단계 — 성과 주기 관리 Admin 화면 ✅
- 평가 주기 목록 (진행중/종료 상태, 목표 수 표시)
- 새 주기 생성 폼 (이름/시작일/종료일, 기존 active 자동 종료)
- 주기 마감 / 재활성화 버튼 (confirm 팝업)
- Flash 메시지 전역 적용 (base.html)
- 사이드바 Admin 전용 "평가 주기 관리" 링크 추가

### 9단계 — 근태 캘린더 팀뷰 개선 ✅
- 오늘 부재 현황 요약 카드 (이름·종류 태그, 부재자 0명 시 안내 메시지)
- 부서 필터 드롭다운 (Admin/Manager 전용, 월 파라미터 유지)
- 셀당 이벤트 최대 3개 + "+N명 더" 축약 표시
- 일/토요일 날짜 색 구분 (빨강/파랑)
- 오늘 날짜 원형 강조 표시

### 10단계 — 모바일 반응형 CSS ✅
- 햄버거 버튼 + 사이드바 슬라이드 오버레이 (768px 이하)
- main-wrapper 사이드바 여백 제거
- stat-grid: 2열 → 1열 (480px 이하)
- grid-2, info-list: 2열 → 1열
- pipeline-board: 가로 스크롤
- 로그인 왼쪽 패널 숨김, 폼 전체 폭
- 캘린더 셀 높이 및 폰트 축소
- 페이지 패딩 28px → 16px
- main.js addEventListener로 토글 처리 (인라인 onclick 미사용)
