Compensation & Payroll 핵심 Gap
결정적으로 없는 것
Salary Band 부재
salary_grades에 단일 annual_salary만 있고 min/midpoint/max 밴드가 없음. Compa-Ratio 계산 불가. 성과등급과 급여 인상이 연결되지 않음 (performance 모듈과 payroll 모듈이 단절).
결정적으로 없는 것
Merit Matrix + ACR 부재
성과등급 × Compa-Ratio 구간 → 인상률 가이드라인 테이블 없음. Annual Compensation Review 프로세스 없음. 매니저가 팀원 인상안을 제안하고 HR이 승인하는 워크플로우 없음.
Workday 패턴 2
Compa-Ratio
Compa-Ratio = 실제연봉 / Midpoint × 100. 80 미만: 하단(Under-paid 위험). 80~120: 정상. 120 초과: Red-circle. admin_payroll 화면에 직원별 Compa-Ratio 컬럼 + 색상 배지 추가.
Workday 패턴 4
ACR (Annual Compensation Review)
HR이 ACR 주기 오픈 → 각 매니저가 팀원 인상안 입력 (Merit Matrix 가이드라인 표시) → HR 검토/승인 → salary_history 기록 + employee_salary 업데이트.

현재 departments + manager_id 데이터는 충분히 갖춰져 있으나 시각화가 없음. SVG 기반 트리 렌더링 (Vanilla JS, 외부 라이브러리 없음). 노드 클릭 → 우측 슬라이드인 패널.
Workday 패턴 5
Compensation Band + Compa-Ratio
salary_grades에 min/mid/max 3개 밴드. 급여 수정 모달에서 밴드 내 현재 위치 슬라이더 시각화. 밴드 초과 시 주황 경고, 미달 시 파란 안내.

Life Event 트리거

Workday 패턴 B
Election Status 4단계 이력
모든 복리후생 선택에 Future/Current/Previous/Historical 상태 유지. benefit_elections 테이블. 급여 계산 시 해당 월의 Current 선택만 참조 → 소급 계산 가능.
