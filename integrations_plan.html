외부 서비스 연동 기획서

Slack · Jira · Confluence 자동화 연동 — HR 이벤트 발생 시 외부 서비스를 자동으로 처리합니다.

💬 Slack 🔵 Jira 📘 Confluence REST API 기반 데모 모드 지원

개요

TalentCore에서 입사·부서이동·퇴직 같은 HR 이벤트가 발생하면, 외부 협업 도구에 자동으로 작업이 생성됩니다. 사람이 수동으로 Slack에 초대하거나, Jira 티켓을 일일이 만들 필요가 없어집니다.

API 없이도 구조 먼저 완성 — Slack·Jira·Confluence API 토큰이 없어도 UI와 연동 로직은 모두 구현합니다. 토큰이 없으면 데모 모드로 동작해서 실제 외부 호출 없이 시뮬레이션 결과를 보여줍니다.

📘 Confluence 연동

Confluence의 팀 페이지를 TalentCore와 동기화합니다. 입퇴사·부서이동이 발생할 때 Confluence 페이지가 자동으로 업데이트됩니다.

📄 팀 멤버 페이지 자동 업데이트
각 팀/부서 Confluence 페이지에 멤버 목록이 있다면, 입사·퇴사·부서이동 시 자동으로 반영됩니다.
🗂️ 온보딩 가이드 자동 공유
신규 입사자 등록 시, 해당 팀의 온보딩 문서 링크를 Slack DM으로 자동 전달합니다.
👤 직원 프로필 페이지 생성
입사 시 Confluence에 개인 프로필 페이지를 자동 생성합니다. (이름·직급·담당 업무·연락처)
📊 조직도 페이지 동기화
TalentCore 조직도와 Confluence 조직도 페이지를 월 1회 자동 동기화합니다.

구현 구조

파일 구조

# 새로 만들 파일들 hr-system/ ├── integrations/ │ ├── __init__.py │ ├── slack.py # Slack API 래퍼 │ ├── jira.py # Jira REST API 래퍼 │ ├── confluence.py # Confluence API 래퍼 │ └── dispatcher.py # HR 이벤트 → 외부 서비스 라우팅 ├── app.py # 기존 HR 이벤트에 dispatcher 호출 추가 └── templates/ └── admin/ └── integrations.py # 연동 설정 페이지

핵심 흐름

# dispatcher.py — HR 이벤트를 받아서 외부 서비스로 분배 def on_employee_created(employee: dict): if slack_enabled(): slack.invite_user(email=employee['email'], name=employee['name']) slack.add_to_channels(email=employee['email'], dept=employee['dept']) slack.post_welcome(employee) if jira_enabled(): jira.create_onboarding_epic(employee) if confluence_enabled(): confluence.add_team_member(employee) def on_employee_terminated(employee: dict): if slack_enabled(): slack.deactivate_user(email=employee['email']) if jira_enabled(): jira.create_offboarding_epic(employee)

app.py 수정 — 기존 코드에 한 줄만 추가

# employee_new() 라우트 맨 끝에 추가 from integrations.dispatcher import on_employee_created # 기존 직원 생성 코드 ... db.execute("INSERT INTO users ...") db.commit() # 연동 트리거 (실패해도 HR 기능은 정상 동작) try: on_employee_created(new_employee_dict) except Exception as e: app.logger.warning(f"Integration error: {e}")

환경 변수 설정

API 토큰은 .env 파일에 넣으면 됩니다. 없으면 데모 모드로 동작합니다.

SLACK_BOT_TOKEN xoxb-로 시작하는 Slack Bot Token (Slack App 설정에서 발급)
SLACK_ADMIN_TOKEN xoxp-로 시작하는 User Token (계정 초대·비활성화용, Pro 플랜 필요)
SLACK_WORKSPACE_ID Slack 워크스페이스 ID (T로 시작)
JIRA_BASE_URL https://your-domain.atlassian.net
JIRA_EMAIL Jira 계정 이메일 (API 인증용)
JIRA_API_TOKEN Atlassian 계정 설정 → 보안 → API 토큰에서 발급
JIRA_PROJECT_KEY 온보딩/오프보딩 이슈를 만들 Jira 프로젝트 키 (예: HR, OPS)
CONFLUENCE_BASE_URL https://your-domain.atlassian.net/wiki
CONFLUENCE_SPACE_KEY 업데이트할 Confluence Space Key (예: TEAM, HR)

데모 모드

API 토큰이 없어도 연동 UI와 로직을 완전히 체험할 수 있습니다.

데모 모드 동작 방식 — 환경 변수가 없으면 실제 API 호출 대신 시뮬레이션 결과를 DB에 기록하고 연동 로그 페이지에 표시합니다. 실제 Slack/Jira/Confluence에는 아무것도 전송되지 않습니다.

구현 로드맵

Phase 1 — 구조 구현 다음 작업
integrations/ 폴더 생성, dispatcher.py, 데모 모드, 연동 로그 DB 테이블, 연동 설정 UI (/admin/integrations), 기존 employee_new·termination에 훅 추가
Phase 2 — Slack 실제 연동
SLACK_BOT_TOKEN 발급 후 invite·채널추가·메시지 발송 실제 연동
Phase 3 — Jira 실제 연동
JIRA_API_TOKEN 발급 후 Epic·태스크 자동 생성, Jira Webhook → TalentCore 체크리스트 동기화
Phase 4 — Confluence 실제 연동
팀 페이지 자동 업데이트, 신규 직원 프로필 페이지 생성
Phase 1만 완성되면 포트폴리오에서 "Slack/Jira/Confluence 연동 구조 구현, API 토큰 발급 시 즉시 활성화"라고 설명할 수 있습니다.
