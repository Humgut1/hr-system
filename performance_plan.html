Workday에서 차용할 핵심 패턴
Workday의 성과관리 철학을 한국 스타트업 맥락에 맞게 적용합니다
패턴 1 — Workday Talent Review
9박스 캘리브레이션 매트릭스
Y축=성과(Performance), X축=잠재력(Potential)으로 직원을 3×3 격자에 배치. 적용: calibration_results에 potential_score(1~3) 컬럼 추가. 매니저가 드래그로 위치 조정 → PATCH /performance/calibration/<id>/position. 각 박스별 직원 카드 렌더링.
패턴 2 — Forced Distribution
등급별 강제 배분 비율
등급별 배분 비율 가이드라인 설정 (예: S 5% / A 20% / B 50% / C 20% / D 5%). 적용: company_config에 grade_distribution_json 컬럼. 캘리브레이션 화면 우측에 현재 배분 vs 목표 배분 바 차트. 비율 초과 시 경고 배지 (soft constraint, 강제 차단 아님).
패턴 3 — Workday Talent Card
직원 종합 Talent Card
성과/역량/포텐셜/경력 포부/후계자 포지션을 한 화면에 집약. 적용: employee_detail.html의 성과 탭을 Talent Card 형식으로 재편. 사이클별 등급 히스토리 타임라인, 현재 9박스 위치 미니 표시, 스킬 태그.
패턴 4 — Workday Succession
후계자 계획 (Succession Planning)
포지션에 대해 Succession Depth(후보 인원수) 정의, Readiness(즉시/1년 내/2년 내) 분류. 적용: succession_plans 테이블 신규 생성. 매니저가 팀원을 특정 포지션 후계자로 등록 가능.
패턴 5 — Merit Matrix
성과-보상 연동
(성과등급 × compa-ratio 구간) → 인상 % 가이드라인 결정. 적용: merit_matrix 테이블 (grade, compa_band, suggested_pct). 캘리브레이션 최종 확정 후 bulk-raise 화면에서 등급별 인상률 자동 제안.

calibration_results 컬럼 추가
-- 9박스 포지셔닝을 위한 potential + perf_band 컬럼 ALTER TABLE calibration_results ADD COLUMN potential_score INTEGER DEFAULT 2; -- 1=Low, 2=Mid, 3=High ALTER TABLE calibration_results ADD COLUMN perf_band INTEGER DEFAULT 2; -- 1=Low(D/C), 2=Mid(B), 3=High(A/S) — final_grade에서 자동 계산 ALTER TABLE calibration_results ADD COLUMN box_position TEXT; -- 예: "3-1" = perf_band 3, potential_band 1 (High Performer, Low Potential) ALTER TABLE calibration_results ADD COLUMN potential_note TEXT; -- 포텐셜 평가 근거 메모
goal_templates 테이블 (신규)
CREATE TABLE goal_templates ( id INTEGER PRIMARY KEY AUTOINCREMENT, title TEXT NOT NULL, category TEXT NOT NULL, -- OKR/KPI/COMPETENCY description TEXT, weight INTEGER DEFAULT 20, job_family_id INTEGER REFERENCES job_families(id), -- NULL이면 전사 공통 position_level TEXT, -- CL1~CL9 또는 NULL created_by INTEGER REFERENCES users(id), created_at TEXT DEFAULT (datetime('now')) );
succession_plans 테이블 (신규)
CREATE TABLE succession_plans ( id INTEGER PRIMARY KEY AUTOINCREMENT, target_position_id INTEGER REFERENCES positions(id) NOT NULL, candidate_user_id INTEGER REFERENCES users(id) NOT NULL, readiness TEXT NOT NULL CHECK(readiness IN ('immediate','one_year','two_years')), nominated_by INTEGER REFERENCES users(id), note TEXT, created_at TEXT DEFAULT (datetime('now')), updated_at TEXT DEFAULT (datetime('now')), UNIQUE(target_position_id, candidate_user_id) );

필수 선행 — Payroll 모듈
Merit 연동(P1-5)은 bulk-raise 라우트 수정 필요. salary_grades에 mid_salary가 정상 입력되어 있어야 Compa-Ratio 계산 가능.
필수 선행 — 직원 모듈
Talent Card, Succession Planning은 employee_detail과 데이터 공유. manager_id, position_id, job_family_id 모두 필요.
독립 가능 — 알림 시스템
P2-4 평가 알림 자동화는 기존 add_notification() 함수 재사용으로 독립적으로 구현 가능.
한국 스타트업 맥락 주의사항
캘리브레이션 참여자 제한: HR + 팀장급만 참여. manager 역할은 자기 팀원 카드만 보이도록 권한 분리 필수.

등급 공개 시점 제어: 전사 캘리브레이션 완료 전까지 개인 등급 비공개. is_shared + calibration_sessions.status='closed' 두 조건 모두 충족 시에만 직원 조회 가능.

포텐셜 평가 기준 명시: 학습 민첩성, 리더십 잠재력, 복잡성 처리 각 1~3점 입력 → 평균으로 potential_score 자동 계산. 임의적 판단 방지.
