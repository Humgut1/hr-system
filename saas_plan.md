## 1. 포지셔닝 (확정)

- **한 줄**: "인사담당자 1명이 100명 관리하는 회사를 위한, 근로기준법 자동 준수 HR"
- **타깃**: 국내 30~200인, 인사 전담 1~2명, 엑셀+수기 운영 중인 회사
- **전략**: Workday 미니어처 ✕ → 컴플라이언스 자동화 SaaS ○. flex와 정면승부하지 않고 "flex는 너무 크고 비싸다"는 회사를 노림
- **차별점 전면 배치**: 52시간 감시·§54 휴게·§56 수당·연차촉진 등 법정 의무 자동화 + AI/MCP 연동(국내 경쟁사에 없음)
- **v1.0 정의**: "모르는 회사 하나가 스스로 가입해서 자기 직원 데이터로 한 달 쓸 수 있는 상태" (기능 추가가 아님)
- **현실 목표**: 유료 10개사가 아니라 **실데이터로 3개월 쓰는 디자인 파트너 1~2개사**

## 2. 요금제 계층 (확정)

| 요금제 | 포함 | 가격(안) |
|---|---|---|
| **Core** (주력) | 인사기록·조직도·근태·52시간·연차·급여명세·증명서·전자결재·문서함 | 인당 월 2,000~3,000원 |
| **Growth** | + 성과·채용(축소형)·온보딩·복지포인트·다면평가 | 인당 월 4,000~5,000원 |
| **Enterprise (숨김)** | 9박스·승계·Merit Matrix·ACR·Talent Card·데이터 마법사 | "문의" 뒤로. 기본 UI에서 숨김 |

- 기존 ₩1,000/인 단일가 폐기 (너무 싸면 급여 데이터 신뢰를 잃음)
- FEATURE_DEFS 기능 게이팅 재활용해서 요금제별 사이드바/메뉴 노출 제어

## 3. 기능별 확정 판정

### 유지 (승헌씨 명시 지시 — 단, 아래 수정 반영)
| 기능 | 수정 방향 |
|---|---|
| 오퍼레터 RSU/사이닝 + 베스팅 차트 | **국내형 스톡옵션 모드 추가**: 주식보상 항목을 "스톡옵션/RSU" 선택형으로, 스톡옵션 선택 시 행사가·클리프 1년(벤처기업법 §16-3 최소 2년 재직 후 행사) 표기. 라벨 한글화 |
| IC/M 투트랙 레벨 (L1~L9, M1~M5) | **직급 체계 프리셋 선택형**: 회사설정에서 "L-레벨형 / 사원-대리-과장-차장-부장형 / 커스텀" 선택. 내부 스키마는 유지, 라벨만 매핑 |
| Jira/Confluence 온보딩 자동화 | 유지하되 **기본 스토리에서 제외**. .env 미설정 시 자동 비활성(이미 그렇게 동작) → "연동 설정 시 활성화되는 옵션 기능"으로 포지셔닝 |
| HR 데이터 마법사 | 유지. 추후 AI 질의(자연어→리포트)와 결합해 차별점으로 승격 후보 |
| SSC 피어리뷰 + 익명성 임계값 | 유지하되 **주기별 토글**: 성과 주기 생성 시 "다면평가 포함 여부" 선택. 미포함 주기에선 관련 메뉴 숨김 |

### 축소/재편
- **전자계약**: 패스 (이번 개편에서 건드리지 않음, 현행 유지)
- **성과관리**: 전면 재개편 (아래 §4)
- **채용 ATS**: 자체 파이프라인 축소 → 외부 ATS 후보자 데이터 수신 + 직원 전환에 집중 (아래 §5)
- **온보딩**: 체크리스트 + 웰컴메일 + 버디 유지 (Jira/Confluence는 옵션 연동으로)
- **복지포인트**: 지급·잔액만 유지, Enrollment Event 구조 제거
- **숨김(Enterprise 격리, 코드 삭제 아님)**: 9박스 캘리브레이션·Potential Score, 후계자 계획, Talent Card·Flight Risk, Merit Matrix, Salary Band/Compa-Ratio, ACR

### 강화 (팔리는 것)
- 근태·52시간·연차·급여명세·증명서·전자결재·문서함 = Core 상품 그 자체
- **연차촉진 알림** (연차촉진제도 법정 절차) 신규
- **급여명세서 이메일/알림톡 발송** 신규 (교부 의무 대응)
- 급여는 "참고용 자동계산 + 담당자 확정" 구조 명시 (세무사 검증 전 정확성 보장 표방 금지)

## 4. 성과관리 재개편 (기획 확정, 구현은 Phase C)

국내 30~200인 기업의 실제 관행 기준 단선 플로우로 재설계:

```
[주기 시작 — HR]
 1. 평가 주기 생성 (반기/연간, 다면평가 포함 여부 토글)
[목표 — 직원+팀장]
 2. 직원이 목표 3~5개 작성 (가중치 합 100%)
 3. 팀장 검토·합의 → 목표 확정 (확정 전 평가 불가)
[상시]
 4. 진행률 업데이트 (분기 체크인 권장 알림)
[주기 말]
 5. 자기평가 (점수+코멘트)
 6. (주기에 포함된 경우만) 다면평가 — SSC 프레임워크, 3명 미만 비공개
 7. 팀장 평가 → 잠정 등급
[확정 — HR]
 8. HR 조정 회의(간이 캘리브레이션): 부서별 등급 분포 확인, 하향 시 사유 필수 + 최대 1단계 (기존 로직 재사용, 9박스 UI는 제외)
 9. 등급 확정 → 본인 공개
[사후]
10. 이의제기 기간 (7일, 신규) → 팀장/HR 재검토 1회
11. 등급별 인상률·상여 배수 자동 연동 (기존 bulk-raise Merit 모드 재사용, Merit Matrix 없이 등급→률 단순 테이블)
```

- 핵심 변경: ①이의제기 단계 신규(국내 실무 필수) ②다면평가를 주기별 옵션화 ③9박스·Potential·Talent Card를 플로우에서 분리(Enterprise) ④단계별 상태머신 명확화 (목표확정→평가중→조정중→확정→이의기간→종료)

## 5. 채용 재편: ATS → "후보자 수신 + 직원 전환" (기획 확정, 구현은 Phase C)

- 자체 칸반/면접/오퍼 파이프라인은 Growth 요금제로 격리 (코드 유지)
- Core 신규 기능 **"입사 예정자"**:
  1. 외부 ATS(그리팅 등)에서 합격자 내보내기 → **CSV 임포트 + 표준 웹훅 수신 엔드포인트** (파트너 API 계약 전엔 CSV가 현실적)
  2. 입사 예정자 목록 (이름/연락처/입사예정일/부서/직급/연봉)
  3. 입사일 도래 → 원클릭 직원 전환 (기존 `from_applicant` 프리필 로직 재사용) → 온보딩 체크리스트·웰컴메일·버디 자동 가동
- 판단 근거: ATS 시장은 그리팅이 무료로 장악 중 → 경쟁 대신 **"어떤 ATS를 쓰든 입사 후는 TalentCore"** 연결 고리 전략

## 6. 실행 순서 (이 순서대로 진행, 하나씩)

### Phase A — 보안 기준선 (v0.96~0.97) ← 지금 여기
1. 웹훅 서명 검증 (토스 `/billing/webhook`, Slack `/slack/command`·`/slack/interactive`) — 시크릿 미설정 시 검증 스킵+경고 로그 구조로 먼저 구현
2. CSRF 토큰 전면 적용
3. 감사 로그 (급여/평가/개인정보 열람·변경 기록)
4. 테넌트 DB 자동 백업 + 복원 절차
5. 비밀번호 정책 (+2FA는 후순위)

### Phase B — 실고객 진입로 (v0.98~1.0)
6. CSV 직원 일괄 임포트 (실고객 진입의 유일한 문)
7. 요금제 3계층 + 사이드바/메뉴 다이어트 (Core 기준 기본 노출 절반 이하)
8. 연차촉진 알림 + 급여명세 발송
9. 테넌트 온보딩 셀프서비스 점검 (가입→설정→임포트→운영)

### Phase C — 제품 재편 (v1.1~)
10. 성과관리 재개편 (§4)
11. 채용 재편: 입사 예정자 기능 (§5)
12. 직급 프리셋 / 스톡옵션 모드 / 다면평가 토글 (§3 유지 기능 수정)
13. 승인 체인 설정화 (결재선 화면 편집)

### Phase D — 스케일 (v1.3~)
14. PostgreSQL 전환 (유료 5개사 전), Effective Dating(조직/급여), 알림 센터 통합, 연말정산

## 7. 메모 (충돌·주의)

- 승헌씨 지시에 표면상 충돌 1건: Jira/Confluence를 "유지" 목록에 넣으면서 "제거 대상" 행에도 "굿" → **"코드 유지 + 옵션 연동化(기본 비활성)"로 합의 해석** (양쪽을 만족). 다르면 정정받을 것
- 코드 삭제는 원칙적으로 하지 않음 — 요금제 게이팅으로 숨김 (데모/포트폴리오 가치 보존)
- 급여 정확성 표방 금지 문구, 개인정보처리방침은 Phase B에서 랜딩/가입 플로우에 반영
